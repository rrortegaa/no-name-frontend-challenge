Parte 1: Conceptos teóricos

    HTML/CSS

      Explica la diferencia entre margin y padding en CSS.

      margin es una propiedad usada para referirnos al espacio que se encuentra fuera del borde de un elemento HTML. Nos permite controlar el espacio entre los bordes de un elemento y otros elementos cercanos. Los márgenes pueden ser positivos o negativos y así aumentar o reducir el espacio entre elementos.

      padding por otra parte, es el espacio que se encuentra dentro del borde de un elemento, entre el contenido del elemento y su borde. Controla el espacio entre el contenido del elemento y su propio borde.

      La diferencia entre margin y padding radica en que el margen controla el espacio fuera del borde del elemento HTML y el relleno se encarga del espacio dentro del borde del elemento.

      ¿Qué es el modelo de caja en CSS y cuáles son sus componentes principales?

      El modelo de caja de CSS es un estandar que describe como deben estructurarse los elementos HTML por el motor de renderizado del navegador en una página web e indica que cada elemento es considerado como una caja rectangular con cuatro áreas esenciales: área de contenido, área de relleno, borde de la caja y margen de la caja. Cada una de estas áreas tiene un papel específico en la disposición y apariencia del elemento en la página.

      content es el área central del elemento y es donde ponemos el contenido real (texto, imágenes u otros elementos HTML). El tamaño del contenido está definido por las propiedades width y height del elemento.

      padding representa el espacio entre el contenido del elemento y su borde. La propiedad padding nos permite agregar espacio adicional a lo ancho y alto alrededor del contenido del elemento.

      border es la línea que establece el límite visual del elemento, el borde rodea el contenido y el relleno del elemento. Para controlar el ancho, estilo y color del borde se usan las propiedades border-width, border-style y border-color respectivamente.

      margin define el espacio entre el borde del elemento y otros elementos cercanos. Con la propiedad margin controlamos la separación a lo ancho y alto entre elementos adyacentes en una página web.

      Otras propiedades que surgieron en un box model alternativo son box-sizing, display y position.

      Para ayudarnos a controlar cómo se calculan las dimensiones de los elementos HTML usamos box-sizing, su valor predeterminado es content-box y para facilitar la consistencia al trabajar con diseños responsivos o de cajas flexibles es mejor utilizar su valor alternativo border-box.

      Con la propiedad display establecemos como se muestra el elemento (como bloque o como línea) y por tanto se comportará de forma diferente dependiendo del valor de display seleccionado.

      position nos permite definir como se posiciona un elemento en el documento, modificando el flujo normal del documento y cómo se superponen los elementos unos sobre otros. Entre sus posibles valores están static, relative, absolute, fixed y sticky

    JavaScript

      Explique el concepto de "hoisting" en JavaScript.

      El termino hoisting se refiere a un comportamiento del motor de JavaScript en el que las declaraciones de variables y funciones son asignadas en memoria durante la compilación, provocando su 'elevación' al inicio del ámbito de ejecución en el que se encuentren. Esto permite que podamos utilizar una función o una variable antes de que sea declarada, sin embargo, el hoisting no aplica a las inicializaciones y estas se quedan en su sitio, así, los valores podrán ser asignados cuando el proceso de compilacion alcance esa instrucción.

      El hoisting es consecuencia del proceso de dos fases que sigue el motor de JavaScript cuando va a ejecutar algún código: la fase de creación donde crea el ámbito de la función y realiza la asignación en memoria de variables y la fase de ejecución en el que ejecuta el código.

      ¿Cuál es la diferencia entre let, const y var en la declaración de variables?

      La palabra reservada var tiene un scope global si se declara fuera de una función y si es declarada dentro de una función, solo es accesible dentro de dicha función, por otro lado, solo es posible acceder a las variables declaradas con let y const dentro del bloque o expresión de código en el que se usan.

      Al usar variables declaradas con var podemos declarar la misma variable tantas veces como lo deseemos, en cambio, let solo nos permite reasignar su valor y, usando const las constantes no se pueden volver a declarar ni se pueden reasignar.

      var además, permite el hoisting a diferencia de let y const que son más seguras.

      Describa cómo funciona el event bubbling en el contexto de eventos de JavaScript.

      event bubbling es un término usado para refererirse a la propogación de eventos desde el lugar en el que se originaron pasando por sus padres y abuelos hasta llegar a la raíz en el DOM.

      Primero, se lanza el evento en el elemento target, a continuación, el evento se propaga desde el elemento objetivo hacia arriba pasando por todos sus ancestros en el árbol DOM, por último, la propagación permite que cada ancestro pueda manejar el evento si está preparado para escuchar el tipo de evento específico que ha sido lanzado.

      Se puede aprovechar esta característica para hacer delegación de eventos y también es posible detener la propagación cuando sea necesario con event.stopPropagation()

    Responsive Web Design

      ¿Qué son los media queries y cómo se utilizan en CSS para crear un diseño receptivo?

      Los media queries son herramientas de CSS que permiten aplicar diferentes estilos a un elemento dependiendo del dispositivo que se utilice, el tamaño del mismo, la resolución de pantalla o la orientación, entre otros; para aplicar los diferentes estilos se basan en condiciones para evaluar el tipo de medio o las características del dispositivo.

      Los media queries surgieron para ayudar a reducir el tiempo de desarrollo, para crear contenido adaptativo y fluido ya que al implementarlos no es necesario crear diferentes páginas web para cubrir los diversos tipos de dispositivos que se utilizan en la actualidad y finalmente, para mejorar la experiencia del usuario sin importar si está accediendo desde un teléfono, una tablet o una computadora.

      Menciona al menos dos técnicas para hacer que las imágenes sean receptivas en una página web.

      Usar unidades relativas para definir el tamaño de la imagen en lugar de pixeles, p.e. vw o vh.

      Utilizar la propiedad max-width de la imagen para hacer que esta se ajuste al ancho disponible en su contenedor pero que no supere el tamaño que le ha sido dado.

      También se puede utilizar la propiedad object-fit de la imagen para controlar como debería redimensionarse para ajustarse a su contenedor.
